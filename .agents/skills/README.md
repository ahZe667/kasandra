# Repo Skills

To jest vendor-neutral warstwa procedur dla pracy agentów i ludzi w repo Kasandra.

## Routing

- mały feature end-to-end: `vertical-slice`
- nowe źródło lub zmiana integracji: `source-integration`
- zmiana SQLite lub SQL: `schema-change`
- synchronizacja dokumentacji i przykładów: `docs-sync`

## Zasady

- Najpierw czytaj `AGENTS.md`, dopiero potem skill.
- Skills opisują procedury pracy, nie zastępują dokumentacji produktowej.
- Jeśli task zmienia kontrakt albo założenia, zsynchronizuj też `docs/` i `examples/`.
